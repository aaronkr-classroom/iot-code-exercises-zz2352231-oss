# chp1 연습문제

1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
센서: 주변 환경의 물리적 상태(온도, 습도, 빛 등)를 감지하여 전기적 신호로 변환하는 입력(Input) 장치입니다. (예: 온도 센서, 초음파 센서)

액추에이터: 전기적 신호를 물리적 동작(회전, 발광, 소리 등)으로 바꾸는 출력(Output) 장치입니다. (예: 서보 모터, LED, 스피커)
  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
환경의 차이: 디바이스(MCU)는 메모리와 전력이 제한적이어서 C/C++나 MicroPython 같은 경량 언어가 적합합니다.
목적의 차이: 대시보드는 복잡한 UI 구현과 데이터 시각화가 중요하므로 웹 기술인 JavaScript(React, Vue)나 서버용 언어를 사용합니다
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
 디바잉스 간 비동기 통신: 기기끼리 직접 연결할 필요 없이 브로커가 데이터를 중계하므로 네트워크 부하가 줄고 연결이 끊겨도 데이터 관리가 용이합니다. (느슨한 결합)

2. Q2
  - c mosquitto와 node-red에대해 설명하고 실행에 적합한 커퓨팅 환경을 제시하십시오
  - Mosquitto: MQTT 프로토콜을 사용하는 오픈 소스 메시지 브로커입니다. 저전력, 저대역폭 환경에서 기기 간 메시지를 전달하는 '우체국' 역할을 합니다.
Node-RED: 흐름 기반의 시각적 프로그래밍 도구입니다. 코딩 없이 노드를 연결하여 IoT 기기, API, 온라인 서비스를 쉽게 통합할 수 있습니다.
적합한 환경: 라즈베리 파이와 같은 싱글 보드 컴퓨터가 가장 적합합니다. 24시간 구동이 가능하며 리눅스 기반으로 두 소프트웨어를 설치하고 운영하기에 최적화되어 있습니다.

3. 3개 이상의 개발 보드를 찾아서 특징과 장단점을 비교하세요.
 Arduino Uno	가장 표준적인 교육용 보드	사용법이 쉽고 방대한 라이브러리 보유	Wi-Fi/블루투스 기본 탑재 안 됨
 ESP32	Wi-Fi 및 블루투스 내장 MCU	가성비가 매우 높고 IoT 구현에 최적	핀 배열이 보드마다 달라 혼란 가능
Raspberry Pi	리눅스 OS가 돌아가는 초소형 PC	고성능, 복잡한 연산 및 서버 운영 가능	가격이 비싸고 전력 소모가 큼

４ 스마트 홈 구축을 위한 아이디어를 다음 조건을 준수하여 정리해보세요. 
  - 전원 제어, 보안, 환경 제어 포함
  - 사용할 센서 목록 작성
  - 대시보드 구성 항목 정리
  - 시스템 통제 시나리오 작성
스마트 홈은 ESP32와 센서를 활용해 구축합니다. 온습도 센서로 에어컨을 자동 제어하고, 조도 센서로 전등 전원을 관리하며, PIR 및 마그네틱 센서로 침입을 감지해 대시보드 알림을 보냅니다. 이를 통해 효율적인 전력 관리와 철저한 보안, 쾌적한 실내 환경을 동시에 실현하는 시스템을 구성합니다.
